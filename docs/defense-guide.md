# MovieFlex 最终答辩材料

本文按 5 人小组、12 分钟陈述加现场演示设计。所有表述基于当前项目代码，不将计划功能表述为已完成。

## 一、项目一句话介绍

MovieFlex 是一个基于 Next.js、MySQL 和 Prisma 的影视内容平台。平台支持影片浏览、搜索与播放、账号认证、收藏和观看历史；管理员可配置 JSON/XML 影视采集源并同步入库；用户播放行为可写入 Kafka，为后续 Spark 离线分析和推荐结果回写提供数据入口。

## 三、12 分钟陈述稿

### 成员 1：背景、目标与前端体验（约 2 分钟）

各位老师好，我们的项目是 MovieFlex，一个面向影视内容浏览、播放和数据分析的全栈平台。

前端采用 Next.js 16、React 19 和 TypeScript。首页数据由服务端从数据库实时读取，展示热门、最新入库、动画和综艺片单。登录用户如果已有推荐结果会看到为您推荐，否则自动降级为热门推荐。

### 成员 2：认证、播放与用户功能（约 2 分钟）

项目使用 Auth.js Credentials Provider 通过 bcrypt 校验密码，用户 ID 和角色写入 JWT 会话。播放时提交开始、进度和结束事件，后端保存观看历史（用户、影片、剧集唯一组合）。

### 成员 3：数据库与接口设计（约 2 分钟）

MySQL 数据库通过 Prisma 操作。影片表以 sourceKey 和 vodId 为联合唯一键，观看历史按用户、影片和剧集区分。接口先校验参数再执行业务逻辑。

### 成员 4：资源采集与后台管理（约 2 分钟）

管理员在后台配置外部采集源（JSON/XML），系统将外部字段统一转换为内部结构，通过分类同义词实现自动映射，无法识别的分类留给管理员审核。

### 成员 5：大数据链路与展望（约 2 分钟）

用户行为事件通过 Kafka 投递，不可用时不影响播放功能。推荐结果读取已接入首页，具体算法由 Spark 任务完成。后续将补充 migrations、后台队列和测试。

## 四、建议现场演示顺序

1. 首页展示热门、分类片单和推荐降级

5. 触发近期采集展示结果

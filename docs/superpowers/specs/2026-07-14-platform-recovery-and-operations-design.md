# 平台恢复与运营后台设计

## 目标

恢复所有已暴露但缺失的用户与后台功能，消除已确认的安全、数据完整性、播放和事件链路缺陷，并将后台完善为统一的蓝白运营控制台。

## 交付顺序

### 批次一：安全与基础闭环

- 实现登录、注册、搜索、个人中心、收藏和观看历史页面，移除所有用户可见死链。
- 登录页仅接受本站相对 `callbackUrl`。
- 公开注册始终创建 `USER`；管理员由显式部署初始化流程创建，不使用“首用户”判断。
- 规范化影片列表的 `page`、`type`、`year` 与排序参数，非法参数不进入 Prisma。
- 对事件 API 施加事件类型、影片 ID、观看进度、字段长度和消息大小验证；匿名事件限流或采样。

### 批次二：采集安全与运营后台

- 后台模块：运营概览、采集源与任务、分类映射、影片库、用户管理、主题管理和数据大屏。
- 采集源支持创建、编辑、启停、删除和任务历史。
- 仅允许可信 HTTPS 采集源；拒绝凭据 URL、回环、内网、链路本地、未指定和多播地址；禁止未验证重定向。
- 每个来源只能有一个活动采集任务；所有模式均限制页面数、总时长、响应体大小与连续失败数。
- 采集任务持久化状态、时间、抓取/保存/失败统计和受限错误样本；仅满足成功条件时更新成功同步时间。
- 分类与映射创建使用事务、唯一约束和冲突重读；映射后台支持审核、忽略和重归类。
- 上游数据写入使用版本保护与非空覆盖规则，旧数据和空字段不能覆盖较新内容。

### 批次三：内容与用户体验

- 影片库支持搜索、编辑、上下架和播放源查看。
- 用户管理支持用户列表、角色变更和行为摘要。
- 主题管理支持选择已安装主题并验证主题资源。
- 播放器完整支持 HLS、直接 iframe 与 resolver iframe；resolver 模式在服务端构造安全 URL。
- 切换播放模式时重置旧错误状态。
- 观看量在实际播放后以时间窗口去重计入，不在详情 GET 渲染时自增。
- 详情页与首页统一使用最新推荐 batch，并按推荐排名稳定排序。

### 批次四：数据链路与数据大屏

- Kafka producer 缓存连接 Promise，避免并发请求使用未连接 producer。
- 行为事件写入数据库 outbox 后再响应，由投递 worker 幂等发送 Kafka；失败可重试并可观测。
- 数据大屏显示 Kafka、Spark、MySQL、最新推荐 batch 和分析任务健康状态；无数据时显示真实空状态。

## 数据与权限边界

- 用户页面仅按当前会话 `user.id` 查询和修改自身数据。
- 管理页面与管理 API 均要求 `ADMIN`；客户端隐藏不是权限控制。
- 采集和事件 API 在服务端验证所有外部输入。
- 不在 URL、日志、响应或前端代码中暴露数据库、认证或基础设施密钥。

## 验收

- 已有导航和鉴权重定向不再进入 404。
- 未授权访问后台跳转到可用登录页；普通用户看到可用权限提示。
- 采集任务无法访问私网或未验证重定向目标，重复触发同来源任务被拒绝。
- 非法列表参数、事件参数和采集参数返回明确 400，不产生 Prisma 500 或静默污染。
- 三种播放器模式各自以正确 URL 渲染，换源后无遗留错误遮罩。
- 构建、类型检查、完整测试和生产 HTTP 检查通过；服务固定监听 `0.0.0.0:3000`。
